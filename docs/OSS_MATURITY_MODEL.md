# OSS Maturity Model (django-mui)

This document defines a simple ladder to measure the maturity of this repository as it moves from an internal phase toward an OSS-ready project.

## Level 0 — Internal / Prototype

- Minimal documentation
- No CI gate required
- Ad-hoc changes allowed

## Level 1 — Governed (Internal Phase Target)

- Protected `main` branch
- All work via feature branches + PRs
- CI workflow runs on PRs
- Basic QA policy exists (see `.governance/qa-policy.yaml`)

## Level 2 — Repeatable Engineering

- Tests exist for critical paths
- Contributor guidelines and code of conduct
- Release notes / changelog process

## Level 3 — OSS-Ready

- Clear licensing and attribution
- Public-facing documentation and examples
- Security reporting process
- Maintained issue and PR templates

## Level 4 — Healthy Community

- Regular releases
- Triage and response SLAs (informal is fine)
- Clear roadmap and governance roles
